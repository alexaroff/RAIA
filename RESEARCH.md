## Оборудование

## Эксперимент №1 — Базовая транскрипция

**Цель:** получить локальную транскрипцию интервью на русском языке.

**Стек:** Python, WhisperX, FFmpeg

**Результат:** успешно. Получены TXT, JSON, SRT. Русский язык поддерживается.

---

## Эксперимент №2 — Переход на WhisperX Turbo

**Цель:** улучшить скорость и качество транскрипции на M2 8GB.

### Проблема 1 — Таймаут при загрузке модели

**Симптом:** `ReadTimeoutError` при загрузке через Hugging Face.

**Решение:** модель скачана вручную через `curl`, размещена локально.

```
~/whisper_models/turbo/
```

---

### Проблема 2 — Конфликт версий библиотек

**Симптом:** `ImportError` — конфликт `transformers` и `huggingface_hub`.

**Решение:** откат версии:

```bash
pip install huggingface-hub==0.34.0
```

---

### Проблема 3 — Конфликт PyTorch и Pyannote

**Симптом:** `_pickle.UnpicklingError` — несовместимость PyTorch 2.8 и Pyannote.

**Решение:** отказ от Pyannote, переход на Silero VAD:

```
--vad_method silero
```

---

### Проблема 4 — Перегрев и нагрузка на M2

**Симптом:** высокая температура, зависания при обработке длинных файлов.

**Решение:** оптимизация параметров запуска:

```bash
export OMP_NUM_THREADS=2
```

Увеличены `batch_size` и `chunk_size`. Добавлена предобработка аудио через FFmpeg перед передачей в WhisperX.

---

### Итог эксперимента №2

**Статус:** ✅ Решено

Рабочий пайплайн транскрипции:

```
Аудио
  ↓
FFmpeg (предобработка)
  ↓
WhisperX Turbo + Silero VAD
  ↓
TXT / JSON / SRT
```

**Цель:** разделить транскрипт на реплики HR и кандидата.

**Метод:** транскрипт вручную передаётся в DeepSeek с промтом на разметку ролей.

**Статус:** 🚧 В работе — результат получается, но процесс ручной.

**Следующий шаг:** автоматизировать через API или локальную LLM.

## Эксперимент №4 — HR-анализ через DeepSeek (облачно, вручную)

**Цель:** получить структурированный HR-отчёт по итогам интервью.

**Входные данные:** Вакансия (описание) + Резюме (CV) + Структурированный диалог (HR / CAND)

**Промт:**

```
Сформируй отчет:
### Общая информация
### Краткое саммари кандидата
### Опыт работы
### Технический стек
### Мотивация к смене работы
### Сильные стороны
### Риски
### Вопросы для следующего этапа
### Итоговая рекомендация

Формат рекомендации:
- Подходит
- Условно подходит
- Не подходит
Обязательно обоснуй вывод.
```

**Статус:** ✅ Работает — отчёт получается качественный, процесс ручной.

**Следующий шаг:** автоматизировать подачу данных и генерацию отчёта.

## Главный технический вызов

Транскрипция решена. Основная нерешённая задача — **автоматическая разметка ролей** без ручного вмешательства.

Необходимо разделять реплики:
- Рекрутер (HR)
- Кандидат (CAND)
- Нанимающий руководитель (HM)

Направления исследования: speaker diarization (акустический подход) vs. semantic role detection через LLM (текстовый подход).
